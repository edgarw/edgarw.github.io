Git的配置

发表于 2016-06-18   |   分类于 软件开发   |  

一、Git支持不同层次的配置文件。按照优先级递减的顺序，如下所示：

  1. .git/config

    版本库特定的配置设置，位于版本库内部，可用--file选项修改，是默认选项。这些设置拥有最高优先级。

  2. ~/.gitconfig

    用户的全局配置设置，可用--global修改。Window下路径是%USERPROFILE%/.gitconfig

    Example:

  3. /etc/gitconfig

    系统范围的配置，如果有UNIX文件写权限，就可以用--system选项修改。这些设置的优先级最低。这个文件可能一开始不存在，不过当你使用命令进行更改是就会出现。它还可能存在于/usr/local/etc/gitconfig路径下，视安装情况。

    Example:

二、其他一些常用的设置命令

  1. 配置你的编辑器

    $ git config --global core.editor emacs

  2. 配置你的比较工具

    $ git config --global merge.tool vimdiff

  3. 查看你的配置 (列出所有配置的内容)

    $ git config --list

  4. 移除配置

    $ git config --unset global user.email

  5. 配置命令别名 (运行了下面的命令后，使用git show-graph就能代替一长串的命令)

    $ git config --global alias.show-graph 'log --graph --abbrev-commit --pretty=oneline'
