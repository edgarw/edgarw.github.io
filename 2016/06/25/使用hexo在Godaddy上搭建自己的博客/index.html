使用hexo在Godaddy上搭建自己的博客

发表于 2016-06-25   |   分类于 网站建设   |  

Background

为了让自己养成写东西的习惯，所以弄了这个博客来记录自己的方方面面。本来想用Jekyll＋Github，后发现Hexo＋Github更加方便。但是搭好之后发现公司的内部网不允许访问Github，而且Github不能建私有的空间，所以去Godaddy弄了个1美元主机来存放Hexo生成的静态页面，然后去Bitbucket建一个账号用来存放Hexo的文件， 这样即使换一台机器也可以随时写博文。

调料包：

  1. Hexo
  2. Godday 1 美元主机
  3. Name.com 域名
  4. Bitbucket（存放Hexo文件）
  5. Dnspod （域名解析）
  6. homebrew (很多大牛极力推荐的osx的包管理工具。通过homebrew命令可以下载、管理很多工具）

一、安装homebrew

Homebrew是什么

linux系统有个让人蛋疼的通病，软件包依赖，好在当前主流的两大发行版本都自带了解决方案，Red hat有yum，Ubuntu有apt-get。如果你用mac os，就没有类似的东东，但是mac os有第三方支持：Homebrew，Homebrew简称brew，是Mac OSX上的软件包管理工具，能在Mac中方便的安装软件或者卸载软件，可以说Homebrew就是mac下的apt-get、yum神器。

Homebrew常用的命令：

  1. 搜索软件：brew search 软件名，如brew search wget

  2. 安装软件：brew install 软件名，如brew install wget

  3. 卸载软件：brew remove 软件名，如brew remove wget

    安装命令

    /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

二、安装node.js

  1. 先检查机子上是不是已经有了node.js

    node -version

  2. 如果没有则安装node.js

    brew install node

三、安装Hexo

  1. 安装Hexo

    npm install hexo-cli -g

  2. 然后进入到你要安装的目录 （目录名可以自己定，我就用我的网站域名）

    cd ~/awkroad

  3. 开始初始化

    hexo init

  4. 开始安装

    npm install

至此安装就已经完毕，非常的简单。如果想在本地看一下效果可以

  1. 生成部署的html文件

    hexo generate

  2. 驱动local server

    hexo server

  3. 然后就可以在浏览器中访问localhost:4000看到最初的效果。具体的hexo使用可以参考网上的教程，非常的具体。

四、create Bitbucket账号并上传源文件

  1. 去Bitbucket申请一个账号
  2. create一个repository，我使用的名字是我的域名－awkroad。
  3. 进入hexo的安装目录删除根目录和主题目录下的.git文件夹

  4. 修改根目录下的.gitignore文件为：

  5. 依次执行以下指令，上传源文件到Bitbucket

这样你的源文件就全部同步到了Bitbucket，下次换机器就可以从Bitbucket上拿了下来改。

$ git clone https://username@bitbucket.org/uusername/respositoryname.git

五、name.com上申请域名

此处略过。

六、申请Godaddy的1美元主机

此处略过，网上大把链接和教程。第一次进cPandel时设置自己申请号的域名比如我的就是awkroad.com

七、去Dnspod上注册并使用域名解析功能

  1. 注册Dnspod
  2. 添加域名
  3. 添加DNS解析记录
    • 首先进入godaddy主机的CPanel界面获取主机的ip地址
    • 然后在Dnspod添加一个A记录指向那个ip地址
    • 再创建一条CNAME记录指向你的域名，主机记录填入www,记录值填入awkroad.com
    • 进入name.com的域名管理页面，设置nameservers，删除默认的，添加dnspod的DNS server，分别是f1g1ns1.dnspod.net和f1g1ns2.dnspod.net
    • 如果你的内容同时分布在两个不同的地方，也可以通过dnspod的国内，国外来区分。只需要加上不同的A记录和CNAME记录。

八、使用FTP工具上传Hexo生成的文件

  1. 在godaddy的cPanel界面创建一个ftp账号，指向/home/username/public_html目录

  2. 通过ftp工具将hexo文件夹下的public目录中的所有文件上传到godaddy的/home/username/public_html目录

  3. 访问awkroad.com就可以看到博客的内容了。
